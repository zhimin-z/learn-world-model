# 架构模式与权衡取舍

你在 P02 构建的 RSSM 就是 RNN 基线——我们将以它为起点，逐步引入五个更新的架构族，通过原理剖析与对比表帮你建立"什么情况下用什么"的直觉。

> 本讲不要求实现这些架构——那是后续项目的工作。重点是理解它们的设计取舍，建立"什么情况下用什么"的判断。

---

## 回顾：你已经有了一个 RNN 基线

P02 的 RSSM 有两条并行路径：

- **确定性路径**（GRU）：$h_t = f_\phi(h_{t-1}, z_{t-1}, a_{t-1})$，捕捉平滑的动态趋势
- **随机路径**：$z_t \sim q_\phi(\cdot \mid h_t, o_t)$，在潜空间里采样当前时刻的不确定性

这个设计在 Dreamer V1/V2 中得到验证，能以很低的计算开销在连续控制任务上取得不错的策略性能。它的局限也很明确：**GRU 的记忆容量随序列变长而衰减**，对需要跨越数百步推理的任务力不从心。

接下来五个架构族，都是为了突破这一限制——只是各自选择了不同的方向。

---

## 架构一：RNN / RSSM（你的基线）

**代表系统**：Ha & Schmidhuber World Models (2018)、Dreamer V1 (2019)、Dreamer V2 (2020)

### 核心机制

GRU（或 LSTM）逐步更新隐状态，每一步的计算开销是 **O(1)**——不管序列有多长，单步推理时间不变。

#### RSSM 的双路径设计

RSSM 在 GRU 之上增加了一个随机潜变量 $z_t$，这个双路径设计绝非随意：

- **确定性路径（GRU）** 负责捕捉世界的"主旋律"——物体的轨迹、速度的惯性、场景的整体趋势。这些规律是平滑的、可预测的，GRU 的线性递归结构非常适合建模它们。

- **随机路径（stochastic latent z）** 负责表达"意外"——同样的动作可能导致不同的结果，世界本身存在随机性。$z_t$ 从一个以 $h_t$ 为条件的分布中采样，让模型能够表示预测的不确定性，而不是强行给出一个单一的"最可能"预测。

两条路径拼接后送入解码器，才能同时捕捉规律与随机性。

#### 为什么 Ha & Schmidhuber 用 MDN-RNN？

在 Ha & Schmidhuber 的原始 World Models (2018) 里，动态模型用的是 **MDN-RNN**（Mixed Density Network + RNN）。为什么不是普通 RNN？

普通 RNN 的输出是一个单点预测——它只能说"下一帧潜向量是 $z^* $"。但真实世界的未来往往有**多个分支**：一扇门可能打开也可能打不开，一辆车可能刹住也可能撞上去。如果强行用单点预测，模型只能输出所有分支的均值——这个"均值预测"在任何分支里都是错的。

MDN-RNN 的解决方案是让网络输出一个**混合高斯分布**的参数（每个高斯的均值、方差和混合权重），从而描述多峰概率分布：

```
普通 RNN：      输入 → 单点预测 z*
MDN-RNN：       输入 → [π₁, μ₁, σ₁, π₂, μ₂, σ₂, ..., πK, μK, σK]
                       ↑ K 个高斯分量的混合，可以表达多峰分布
```

RSSM 进一步把这个想法系统化：不是每步输出一个分布参数，而是直接在计算图里维护一个随机路径，让不确定性成为模型的一等公民。

#### 梦境训练（Dream Training）的直觉

Dreamer 的训练流程分三个阶段，每个阶段冻结不同的模块：

```
阶段1：训练 World Model（V + M）
       真实轨迹 → 编码器 V → 潜向量 z
       z + a → RSSM (M) → 预测下一帧 z'
       最小化 ELBO 损失

阶段2：在"梦境"里训练 Controller (C)
       RSSM 生成想象轨迹（纯latent，不调用真实环境）
       Controller 在 latent 空间里学习策略

阶段3：把策略迁移到真实环境，用少量真实交互微调
```

这个流程的核心直觉是："**先学世界，再学行动。**" 世界模型学会物理规律后，Controller 可以在虚拟梦境里用极低的成本做大量探索——一次真实采样可以产生数百条想象轨迹用于训练。这也是为什么 Dreamer 在样本效率上远超无模型 RL 的根本原因。

### 优势

- **计算开销极低**：适合实时控制场景，单步延迟以毫秒计
- **训练稳定**：ELBO 目标明确，易于调试
- **与 actor-critic 天然契合**：Dreamer 直接在潜空间展开想象轨迹，用于策略梯度更新

### 劣势

- **长时记忆弱**：梯度在长序列上衰减，难以建模数百步以上的依赖关系
- **预测精度有限**：生成质量不如 Diffusion，语义抽象能力不如 JEPA

### 适用场景

简单到中等复杂度的连续控制任务（如 DMControl、Atari），对延迟敏感的在线强化学习。

## 架构二：Transformer-based（2021—2022）

**代表系统**：STORM (2023)、IRIS (2022)

### 核心机制

用 **Transformer** 替换 GRU，将历史观测序列 $o_{1:t}$ 分词为离散 token，然后用**自注意力机制（self-attention）**在整个序列上计算权重：

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V$$

> **📖 自注意力中的 Q、K、V**：每个序列位置的向量被线性变换为三个角色——**Query（查询，Q）**：当前位置想"问什么"；**Key（键，K）**：其他位置"提供什么信息"；**Value（值，V）**：实际携带的信息内容。$QK^\top$ 计算每对位置之间的相关性得分，除以 $\sqrt{d_k}$（$d_k$ 为向量维度，防止点积过大导致 softmax 梯度消失），再用 softmax 归一化为注意力权重，最后加权求和 $V$，得到每个位置的输出。直觉上：每个位置都在"问"（Q）其他所有位置，哪些位置的答案（K）与我相关，然后按相关性加权提取它们的内容（V）。

每个位置都能直接"看到"序列里任意一个历史时刻，不再受限于 GRU 的隐状态瓶颈。

#### VQ 离散化：把图像变成"句子"

IRIS 的一个关键操作是 **VQ-VAE 量化**——把连续的图像帧变成离散 token 序列。为什么要这样做？

类比语言：GPT 能预测"下一个词"，因为词是离散的、有限的，概率分布可以用 softmax 精确建模。如果我们能把图像也变成类似"词"的离散单元，就可以直接用 GPT 风格的自回归 Transformer 预测"下一个视觉词"。

VQ-VAE 正是做这件事：它维护一个**"词表"（codebook）**——一组固定数量的可学习向量（如 512 个），把每个图像块的连续编码向量替换为词表中**欧氏距离最近**的那个向量的索引（index），从而把图像变成一串离散 index：

> **📖 VQ（向量量化）的工作原理**：①编码器将图像块映射为连续向量 $z$；②在 codebook 中找到与 $z$ 最近的向量 $e_k$（$k = \arg\min_j \|z - e_j\|_2$）；③用 $e_k$ 的索引 $k$ 代替连续向量，传入 Transformer。反向传播时用**直通估计器（straight-through estimator）**：前向传播用量化后的离散向量，反向传播时假装量化操作不存在，梯度直接流过（把 $e_k$ 的梯度复制给 $z$）。这样既保留了离散表示的好处，又让编码器可以端到端训练。

> **📖 自回归预测（autoregressive prediction）**：每次只预测序列中的下一个元素，并以之前所有元素为条件——$P(\text{token}_t | \text{token}_1, \ldots, \text{token}_{t-1})$。生成时，先预测 token$_1$，再以 token$_1$ 为条件预测 token$_2$，以此类推。GPT 就是这样生成文字的；IRIS 用同样的方式生成图像 token 序列。

```
原始帧 (64x64 像素)
    ↓ VQ-VAE 编码器
图像块 → 词表索引序列 [42, 7, 183, 56, ...]  ← 像句子一样
    ↓ Transformer
预测下一帧的 token 序列 [91, 12, 204, ...]
    ↓ VQ-VAE 解码器
重建图像
```

这个类比让世界模型的问题变成了"图像语言建模"，从而直接复用 NLP 领域大量成熟的 Transformer 架构和训练技巧。

#### STORM 的核心改进

STORM（Stochastic Transformer-based wORld Models）并非简单地把 GRU 换成 Transformer，它做了一个更精细的融合：

- 保留 RSSM 的随机 latent $z_t$（不确定性表达）
- 用 Transformer 替换 GRU（长程依赖）
- 添加随机 token 预测目标（让 Transformer 学会建模不确定性，而不只是确定性预测）

这意味着 STORM 不是纯粹的自回归视频模型，而是**带动作条件的随机世界模型**——动作 $a_t$ 作为额外 token 拼接进序列，预测的是动作条件下的未来 latent 分布，而非像素序列。

#### IRIS 的 world model as language model 直觉

IRIS 走得更彻底：整个 Atari 游戏帧被编码为离散 token，世界模型就是一个纯粹的 Transformer 语言模型，在 token 序列上做自回归预测，连奖励和终止信号也被建模为特殊 token。这让世界模型的训练完全复用语言模型的技术栈，包括 KV 缓存、Flash Attention 等工程优化。

### 优势

- **长程依赖建模能力强**：自注意力的感受野覆盖整个上下文窗口
- **支持并行训练**：Transformer 天然并行，训练速度远超 RNN
- **逻辑推理和规划更精准**：适合需要跨多步因果推理的游戏或决策任务

### 劣势

- **计算量随序列长度二次增长**：$O(T^2)$ 注意力，长序列时显存和时间开销陡增
- **推理延迟比 RNN 高**：不适合对实时性要求极高的场景
- **需要更多数据才能收敛**：Transformer 参数量大，样本效率不如 RNN

### 适用场景

复杂游戏（Atari 长游、策略游戏）、需要多步规划的任务；有充足算力和数据时的首选。

## 架构三：Diffusion-based（2023—2024）

**代表系统**：Diamond (2024)、Sora (OpenAI, 2024)、GameNGen (Google, 2024)

### 核心机制

扩散模型通过**逐步去噪**生成输出：先向真实帧添加高斯噪声，再训练网络预测噪声（或直接预测干净帧）：

$$p_\theta(x_{t-1} \mid x_t) = \mathcal{N}(x_{t-1};\, \mu_\theta(x_t, t),\, \sigma_t^2 I)$$

在世界模型场景中，以历史帧和动作为条件，扩散模型逐步"去噪"出下一帧。

#### 去噪过程的直觉

扩散世界模型不是一步生成下一帧，而是从纯噪声出发，一步步把它"洗干净"：

```
纯噪声帧 x_T
    ↓ 去噪步骤 1（以动作 a_t 和历史帧 o_{1:t} 为条件）
x_{T-1}（稍微清晰一些）
    ↓ 去噪步骤 2
x_{T-2}
    ...
    ↓ 去噪步骤 N
x_0（清晰的预测帧）
```

每一步去噪都是一次完整的神经网络前向传播，网络在"动作条件"的引导下决定"把哪里的噪声去掉"。最终结果是一帧在视觉上极度真实的预测，因为扩散过程本质上是在学习图像的自然统计分布。

#### GameNGen 的意义

GameNGen (2024) 是第一个用神经网络**实时**运行完整游戏引擎的系统——它以 20fps 的速度模拟《毁灭战士》(DOOM)，玩家的每一次射击、每一次转身都由神经网络即时预测，而不是由传统游戏代码驱动。

这件事的意义超越了技术细节：**模型本身就是游戏引擎**。这意味着，在原始代码不可得（或太复杂）的情况下，你可以通过观察行为数据"学出"一个游戏引擎——这对模拟器构建、物理引擎近似乃至科学仿真都有深远的影响。

#### 为什么慢？

每生成一帧，扩散模型需要 10–100 步去噪迭代，每步都是一次完整的 U-Net 前向传播（对大模型而言可能涉及数十亿次乘加运算）。相比之下，GRU 只需一步矩阵乘法。

这个延迟结构导致扩散世界模型在**在线 RL 训练循环**里非常昂贵：如果要生成 1000 条想象轨迹，每条 50 步，每步 20 次去噪，那就是 100 万次神经网络前向传播——这在工程上是不可接受的。

### 优势

- **视觉真实度极高**：生成的帧在细节、光照、纹理上远超 GRU/Transformer 解码器
- **泛化能力强**：大规模预训练后可迁移到多个环境
- **能建模复杂的多模态分布**：不同于 VAE 的单峰假设，扩散模型天然支持多模态输出

### 劣势

- **推理慢**：一帧通常需要 10–100 步去噪迭代，难以达到实时控制频率
- **难与策略优化直接对接**：逐步采样过程不可微，反向传播穿越扩散链非常困难
- **计算资源消耗巨大**：训练和推理都比前两类架构贵一个数量级以上

### 适用场景

离线视频预测、高保真仿真器、影视/游戏内容生成；不适合需要实时闭环控制的 RL 场景。

## 架构四：JEPA（2023，非生成式）

**代表系统**：I-JEPA (2023)、V-JEPA (2024)、V-JEPA 2 (2025)，由 Yann LeCun 主导提出

### 核心机制

JEPA（Joint Embedding Predictive Architecture）的核心理念是：**不预测像素，在语义潜空间里预测**。

给定当前观测 $x$，编码器将其映射到语义表示 $s_x$；预测器根据上下文预测目标区域的表示 $s_y$，而非重建像素 $y$：

$$\hat{s}_y = f_\theta(s_x,\, \text{context})$$

训练损失直接在表示空间计算（如 $\ell_2$ 或余弦距离），无需解码回像素空间。

#### 为什么不预测像素？

像素空间充满了与任务无关的信息：光照变化、纹理细节、阴影方向、传感器噪声。一个像素级重建模型必须把模型容量花在学习"在这个光照角度下这块皮肤的纹理应该是什么颜色"上——而这对理解"这只手是否握住了杯子"毫无帮助。

更根本的问题是：**像素空间预测的优化目标和语义理解的目标天然冲突**。均方误差会让模型输出模糊的"平均图像"（因为模糊的图像往往比清晰但错误的图像有更低的 MSE）；GAN 可以生成清晰图像，但引入了训练不稳定性。JEPA 的回答是：**根本不进入像素空间，直接在语义层面预测**。

#### context encoder + predictor + target encoder 三件套

```
输入帧 x（例如一段视频片段）
    |
    ├─── context encoder (f_θ) ──→ context 表示 s_x
    |        ↑                          ↓
    |        (在训练中更新)          predictor (g_φ)
    |                                   ↓
    └─── target encoder (f_ξ) ──→ target 表示 s_y
             ↑                     ↑（目标：predictor 预测与此匹配）
             (EMA 慢更新，不反传梯度)

损失 = || predictor(s_x) - stop_gradient(s_y) ||²
```

这里有一个关键的工程细节：**target encoder 使用 EMA（Exponential Moving Average，指数移动平均）更新**，而不是直接反传梯度。

> **📖 stop-gradient（停止梯度）**：在损失公式 `|| predictor(s_x) - stop_gradient(s_y) ||²` 中，`stop_gradient(s_y)` 表示对 $s_y$ 的计算**不参与反向传播**——梯度在此处被截断，不会流回 target encoder。这样 target encoder 只被 EMA 缓慢更新，而不被当前损失直接拉动。
>
> **📖 EMA（指数移动平均）**：参数更新规则为 $\xi \leftarrow \tau \xi + (1-\tau) \theta$，其中 $\theta$ 是 context encoder 的参数，$\xi$ 是 target encoder 的参数，$\tau \approx 0.996$ 是动量系数（接近 1）。效果是 target encoder 以极慢的速度"跟随"context encoder 更新，始终比 context encoder 滞后，提供稳定的预测目标。
>
> **📖 表示坍缩（representation collapse）**：一种退化解——如果不加约束，神经网络可能发现"把所有输入都映射到同一个向量"是最小化某些损失的捷径（对比损失为零，但表示完全无信息）。EMA + stop-gradient 的组合通过让两个编码器异步更新，破坏了产生坍缩的对称性，使模型必须学到有意义的表示。

这是为了防止"表示坍缩"——如果两个编码器都自由更新，最简单的"解"是让所有输入都映射到同一个向量（损失为 0，但没有任何信息）。EMA 更新让 target encoder 提供一个相对稳定的"参考答案"，从而避免坍缩。

#### V-JEPA 2 的定位

Meta 在 2025 年发布 V-JEPA 2 时，明确把它定位为"**迈向 AGI 的世界模型组件**"，而不是视频生成器。V-JEPA 2 能在给定动作序列的情况下，在语义空间预测未来的视觉表示——这正是具身智能所需要的：不是生成逼真的视频，而是理解"如果我这样移动手臂，物体会在哪里"。

这个定位的转变很重要：JEPA 不参与"谁能生成更逼真的视频"的竞争，它的目标是"谁能更好地理解物理世界"。

### 优势

- **鲁棒、高效**：不重建无关像素，表示更紧凑，对光照/纹理变化不敏感
- **训练成本低**：不需要像素级解码器，无需对抗训练
- **与下游任务对齐**：学到的表示天然面向语义理解，迁移性好

### 劣势

- **不产生可视化输出**：无法生成下一帧图像，难以做视觉质量评估
- **评估指标非直观**：潜空间预测误差不如像素 MSE/FID 直观可解释
- **控制/规划研究尚不成熟**：如何基于 JEPA 表示做 MPC 或 actor-critic，仍是开放问题

### 适用场景

视觉表示预训练、语义相似性任务、数据高效的下游分类/检索；未来有望成为通用世界模型基础。

## 架构五：Robotic World Model（RWM）——机器人控制的硬问题

**代表系统**：Self-Forcing (2024)、RWM-U (2025)

前四个架构族的主要战场是"生成质量"或"游戏智能"。但机器人控制领域有一类更"硬"的世界模型问题，核心不是"能不能生成逼真的图像"，而是"能不能训练出真实可部署的 policy"。

### 两个核心问题

#### 问题一：long-horizon rollout 不发散

在机器人训练中，policy 需要在世界模型里展开数十甚至数百步的想象轨迹来学习。但现有世界模型在多步预测时有一个根本弱点：**误差会积累**。

训练时，模型每步都以**真实状态**作为输入（teacher forcing）：
```
训练（teacher forcing）：
  s_1(真实) → 预测 s_2'
  s_2(真实) → 预测 s_3'   ← 每步都用真实状态，误差不积累
  s_3(真实) → 预测 s_4'
```

但推理时，模型必须以**自己的预测**作为输入（autoregressive rollout）：
```
推理（autoregressive）：
  s_1(真实) → 预测 s_2'
  s_2'(有误差) → 预测 s_3''  ← 误差开始积累
  s_3''(误差更大) → 预测 s_4'''  ← 轨迹发散
```

这个训练与推理之间的分布差距（train-inference discrepancy）导致长程 rollout 迅速偏离真实轨迹，最终产生物理上不可能的状态——policy 在"幻想世界"里学到的策略迁移回真实环境后往往失效。

#### 问题二：policy exploitation

即使世界模型短期预测还算准确，policy 也会主动寻找并利用模型的错误。这个现象叫 **policy exploitation**：policy 会发现某些动作序列在世界模型里能产生虚假的高奖励，但在真实环境里这些动作毫无意义甚至有害。这是所有基于模型的 RL 方法都面临的共性问题，在机器人控制里尤为突出。

### Self-Forcing：让模型习惯自己的预测

**Self-Forcing** 的思路是在训练时就"模拟"推理时的误差积累：不总是喂给模型真实状态，而是有时候喂给它自己上一步的预测，并在**多个步骤**上同时计算与真实状态的损失。

```
Self-Forcing 训练：
  s_1(真实) → 预测 s_2'
  s_2'(自身预测) → 预测 s_3'  ← 用自己的预测继续 rollout
  s_3'(自身预测) → 预测 s_4'
  损失 = sum( ||s_i' - s_i(真实)||² )  for i=2,3,4  ← 每步都和真实状态算 loss
```

这样训练出来的模型对自身预测误差有更强的鲁棒性——因为它在训练时就已经见过"基于有误差输入的预测"，而不是总在理想条件下学习。实验结果表明，Self-Forcing 能将 50 步 rollout 的累积误差降低到 teacher forcing 的约 1/3。

### RWM-U：用 ensemble uncertainty 约束 policy

**RWM-U**（Uncertainty-Aware Robotic World Model）的思路不同：与其让世界模型本身更准确，不如让 policy 知道什么时候世界模型**不可靠**，并主动避开那些区域。

具体做法是训练一个**模型集成（ensemble）**：同时训练 N 个（通常 5–10 个）独立初始化的世界模型，用它们预测的**方差**作为不确定性的估计。

> **📖 认知不确定性（epistemic uncertainty）**：不确定性有两种来源。**偶然不确定性（aleatoric）**来自环境本身的随机性（掷骰子的结果永远无法完全预测），无法通过更多数据消除。**认知不确定性（epistemic）**来自模型"见过的数据不够多"——在训练数据覆盖充分的区域，多个独立模型会给出相近的预测（方差小）；在训练数据稀少的区域，各模型会给出分歧较大的预测（方差大）。模型集成的方差正是认知不确定性的简单估计：如果 5 个模型对同一输入的预测差异很大，说明这是一个"没见过的状态"，world model 在这里不可信。

在 policy 优化时，对高不确定性区域施加惩罚：

```
state + action
    ↓
[世界模型 1, 世界模型 2, ..., 世界模型 N]  ← 集成
    ↓
预测 [s'_1, s'_2, ..., s'_N]
    ↓
方差 Var(s'_1..N) = epistemic uncertainty
    ↓
policy 奖励 = 任务奖励 - λ × uncertainty  ← 惩罚高不确定性区域
    ↓
policy 学会在模型可靠的区域内探索
```

整个系统的工作逻辑是：让 policy 在世界模型里学习，但只在模型"有把握"的地方行动，避免跑进"幻想区域"（模型集成分歧大、不确定性高的状态空间角落）。

### RWM 这条线的价值

把 RWM 单列出来作为一个架构族，是因为它代表了一个不同的评价标准：**世界模型的好坏不只看生成质量，还要看能否训练出真实可部署的 policy**。这两个目标有时一致，有时冲突。Self-Forcing 和 RWM-U 分别从"提高模型鲁棒性"和"约束 policy 行为"两个方向回答了同一个问题：如何把世界模型从"有趣的研究工具"变成"可靠的机器人训练基础设施"。

### 适用场景

高频机器人控制（关节空间 MPC、灵巧操作）、对 sim-to-real 迁移要求严格的任务；state-based 动力学建模而非像素级预测。

## 架构六：从 World Model 到 World Action Model（WAM）

**代表系统**：Motus (2025)、DreamDojo (2025)、DreamZero / WAM (2025—2026)

这是 2025–2026 年最活跃的前沿方向。研究者开始问一个更根本的问题：世界模型和策略模型，真的需要是两个分开的模块吗？

### 三个概念的区分

```
World Model:   observation + action → future observation / state
VLA:           observation + language → action
WAM:           observation + language → future observation + action
```

传统的 World Model 以动作为输入、预测未来状态，是 policy 旁边的一个 simulator。

VLA（Vision-Language-Action model）绕过了世界模型，直接从视觉和语言指令预测动作，是一个端到端的 reactive policy。

WAM（World Action Model）试图同时做两件事：预测世界的未来状态，同时预测应该采取的动作。世界的视觉演化成为动作学习的 dense supervision，而不只是一个辅助任务。

### Motus：统一的 latent action world model

Motus 的核心贡献是引入了 **latent action** 的概念：从异构视频数据（包括大量没有动作标签的人类视频）中自动抽取动作表征。

```
无标注视频帧序列 [o_t, o_{t+1}, o_{t+2}, ...]
    ↓ latent action encoder
latent action sequence [a_t^lat, a_{t+1}^lat, ...]  ← 自动发现的动作表征
    ↓ 同时训练以下目标
    ├─ 视频生成（给定 latent action，预测下一帧）
    ├─ 逆动力学（给定相邻帧，预测 latent action）
    └─ 动作预测（映射 latent action 到真实机器人动作）
```

这个设计的优势是：可以用互联网上海量的无标注人类视频来训练世界模型，然后用少量有机器人动作标签的数据来做"对齐"——大幅降低了数据采集成本。

### DreamDojo：从第一视角人类视频预训练

DreamDojo 的出发点是接触丰富的灵巧操作（contact-rich dexterous tasks）——这类任务极难用仿真器生成真实数据，但人类做这些动作的视频相对容易获得（洗碗、做饭、整理物品等第一视角视频）。

核心设计是 **continuous latent actions**：不把动作离散化为按键或关节角度，而是学习一个连续的低维动作空间，使得在这个空间里插值对应视频里真实可能的动作序列。这样即使没有精确的动作标签，模型也能从纯视频里学到"有效的动作表征"。

训练流程：
```
大规模第一视角人类视频（无动作标签）
    ↓ 预训练：学习 continuous latent action space
latent action world model（能预测未来视频帧）
    ↓ 少量机器人演示数据（有动作标签）
fine-tune：把 latent action 对齐到机器人关节空间
    ↓
可部署的灵巧操作 policy
```

### DreamZero / WAM：视频作为 dense supervision

DreamZero 和 WAM 系列论文的共同思路是：用预训练的 **video diffusion backbone** 同时预测未来世界状态和机器人动作，相比 VLA 只做 action regression，WAM 用视频序列作为 dense supervision。

```
VLA 的监督信号：
  observation → [action_1, action_2, ..., action_T]
  只有动作序列这一种 loss

WAM 的监督信号：
  observation → [future_frame_1, future_frame_2, ...] + [action_1, action_2, ...]
  视频生成 loss + 动作 loss，相互增强
```

为什么视频预测有助于动作学习？因为如果模型能准确预测"我推这个杯子后它会怎么移动"，它对物理世界的理解必然比只预测动作的模型更深——而这个深层理解会反哺动作质量的提升。

### 这批论文揭示的新趋势

**world model 不再只是 policy 旁边的 simulator，而是 policy 本身的一部分。**

在传统的 model-based RL 框架里，world model 和 policy 是两个分离的模块：world model 提供想象轨迹，policy 在轨迹上优化。WAM 系列论文正在打破这个分离：它们训练的是一个**统一模型**，同时建模世界动态和决策逻辑，两者共享同一套表示，相互增强。

这个趋势的背后逻辑是：对世界未来的预测（world modeling）和对行动序列的规划（action planning）共享同样的因果结构——理解"发生了什么"和"应该做什么"本质上是同一个认知能力的两个侧面。

## 对比总结表

| 架构族 | 核心优势 | 主要劣势 | 典型适用场景 |
|--------|----------|----------|--------------|
| **RNN / RSSM** | 计算开销低、延迟小 | 长时记忆弱、生成质量有限 | 在线 RL、实时控制、资源受限设备 |
| **Transformer** | 长程依赖强、并行训练快 | 计算量随序列二次增长 | 复杂游戏、多步规划、数据充足场景 |
| **Diffusion** | 视觉真实度极高 | 推理慢、难实时控制 | 离线仿真、高保真视频生成 |
| **JEPA** | 鲁棒高效、忽略无关噪声 | 无像素输出、控制应用尚不成熟 | 语义表示预训练、迁移学习 |
| **RWM** | 长程 rollout 稳定、policy 不漂移 | 计算开销高（集成），主要适用 state-based 任务 | 机器人高频控制、sim-to-real 迁移 |
| **WAM** | 世界预测与动作规划联合优化 | 架构复杂、数据需求大 | 具身智能、灵巧操作、语言条件机器人 |

## 如何选择架构？

面对一个新任务，可以沿以下路径做决策：

```
任务是否需要实时闭环控制？
├─ 是 → 是否为机器人高频控制？
│        ├─ 是，且需要 sim-to-real 可靠性 → RWM（Self-Forcing / RWM-U）
│        └─ 否（游戏/模拟环境）→ 序列长度要求高吗？
│                 ├─ 否（< 100步）→ RNN / RSSM（你的 P02 基线）
│                 └─ 是（需要长程规划）→ Transformer（STORM/IRIS）
└─ 否 → 最终目标是什么？
         ├─ 视觉保真度（仿真器/视频生成）→ Diffusion（Diamond/Sora）
         ├─ 语义推理（表示学习/迁移）→ JEPA（I-JEPA/V-JEPA）
         └─ 具身智能（动作+世界联合建模）→ WAM（Motus/DreamZero）
```

**实践建议**：

从 RNN/RSSM 起步——它实现简单、调试容易，P02 已经帮你走完这一步。遇到瓶颈再升级：长序列预测精度持续下跌、或任务需要跨多步因果推理，再考虑切换 Transformer。Diffusion 留给离线场景，别在在线 RL 训练循环里用。JEPA 目前控制接口尚不成熟，但表示学习和迁移任务已经有实质结果，值得跟踪。做真实机器人的话，Self-Forcing 和 ensemble uncertainty 这类工程手段比换架构更重要——先把长程稳定性解决掉，架构是次要问题。

## 关键概念速查

| 术语 | 含义 |
|------|------|
| **RSSM** | 循环状态空间模型，Dreamer 的核心动态模块，含确定性（GRU）和随机路径 |
| **MDN-RNN** | 混合密度网络 RNN，输出多峰概率分布而非单点预测，用于表达未来多分支不确定性 |
| **梦境训练** | Dreamer 的训练策略：先冻结 World Model，让 Controller 完全在 latent 想象轨迹里训练 |
| **VQ 离散化** | 用向量量化把连续图像帧映射到离散 codebook 索引，使 Transformer 可做自回归预测 |
| **自注意力** | Transformer 的核心算子，让每个位置直接关注序列中所有其他位置，计算复杂度 O(T²) |
| **去噪扩散** | 通过反复预测并去除噪声来生成样本的概率生成框架 |
| **JEPA** | 联合嵌入预测架构，在语义潜空间做预测，不重建像素 |
| **Self-Forcing** | 训练时让模型基于自身预测继续 rollout，并在多步上计算 loss，缓解长程漂移 |
| **ensemble uncertainty** | 训练多个独立模型，用预测方差估计 epistemic uncertainty，引导 policy 避开不可靠区域 |
| **latent action** | 从无标注视频中自动抽取的低维动作表征，无需精确动作标签即可训练动作感知世界模型 |
| **WAM** | World Action Model，同时建模世界未来状态和行动序列的统一模型，相比 WM+policy 分离架构 |
| **RWM** | Robotic World Model，专注于机器人控制场景的世界模型，重点解决长程 rollout 稳定性和 policy exploitation 问题 |
| **潜空间漂移** | 多步预测误差积累导致潜表示偏离真实分布的现象，所有架构族都需面对 |
