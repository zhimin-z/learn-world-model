# Lecture 04：学习范式与规划方法

> Part A 解决"数据从哪里来、模型怎么被训练"，Part B 解决"有了模型，怎么用它做决策"。两部分顺序读效果最好。

---

## Part A：学习范式（先问"为什么"）

不同的世界模型，其训练数据的来源和形式截然不同。这催生了三种本质不同的学习范式。理解它们，就是理解一个模型的"知识边界"。

---

### 范式一：观察型（Observation-only）

**核心思想**：只看视频或图像序列，完全不涉及动作信号。模型学到的是世界的视觉规律，而非动作-结果的因果关系。

**数据来源与规模**：互联网视频（YouTube、电影、游戏录像）是这类方法最重要的数据来源。其规模是机器人收集的交互数据的几个数量级——机器人每小时采集几百条轨迹，而互联网上每分钟上传超过500小时的视频。这种海量的被动数据正是驱动 Sora 和 Genie 这类大规模世界模型的燃料。

**代表系统**：
- **Sora**（OpenAI，2024）：在海量互联网视频上训练，能生成符合物理规律的长视频片段。
- **Genie 1**（Google DeepMind，2024）：从无动作标签的2D平台游戏视频中，通过自监督学习抽取帧间的"潜在动作（latent action）"表征。这使被动视频变成了可交互的世界——给定一张静态截图，Genie 可以让它变成一个玩家可以操控的互动环境，即使原始数据中从来没有真正的动作标签。
- **Genie 2**（Google DeepMind，2024）：大规模基础世界模型（large-scale foundation world model）。只需输入一张图片，Genie 2 就能生成一个可被人类或 agent 操作的3D环境。世界模型从"预测工具"升级为"agent 的训练场"——agent 无需接触真实环境，就能在 Genie 2 生成的虚拟世界里学习技能。

> **💡 直觉**：Genie 的 latent action 机制解决了一个经典难题：如果视频里没有动作标签，我们还能从中提取"谁在控制什么"吗？答案是肯定的。相邻两帧之间的差异本身就隐含了"某种控制信号"。自监督学习通过最小化预测误差，迫使模型将这个隐含信号编码为结构化的 latent action 向量。

**数学形式**：模型学习的是条件分布

$$p(o_{t+1} \mid o_{\leq t})$$

而不涉及任何动作 $a_t$。

**适用场景**：
- 场景视频生成与世界仿真
- 数据增强（为下游强化学习生成训练数据）
- 无动作的预测性建模

**局限性——被动预测 vs 主动控制**：观察型范式能"看"未来，但不能"控制"未来。被动视频预测学的是"世界按自然规律如何演化"，而机器人真正需要的是**反事实推理（counterfactual reasoning）**：**如果我换一个动作，世界会怎样？**

> **📖 反事实推理**："反事实"（counterfactual）字面意思是"与事实相反的假设"。反事实推理是指在已知"实际上发生了 A"的情况下，推断"如果当时选择了 B，会发生什么"。对机器人来说，这意味着能回答"如果我的手臂向左而不是向右移动，杯子会在哪里"——这需要一个包含动作条件的世界模型，而不仅仅是被动的视频预测。

没有动作条件的转移模型，无法回答这个问题。这是观察型范式与交互型范式之间最本质的鸿沟。

### 范式二：交互型（Interaction-based）

**核心思想**：智能体亲身与环境交互，收集 $(o_t, a_t, r_t, o_{t+1})$ 四元组，通过真实的"试错"来学习动作-结果的映射。

**数据格式说明**：交互型方法收集的不只是"成功轨迹"，还包括失败、试探、碰撞和恢复轨迹。这一点至关重要——world model 需要知道"做什么会发生什么"，而不只是"怎么做对"。一个只见过成功轨迹的模型，无法预测机器臂碰撞桌面后会发生什么，也就无法在规划时避免这种失败。

**代表系统**：
- **Dreamer 系列**（Hafner et al., 2019—2023）：在真实环境中收集少量数据，在想象轨迹中大量训练策略。Dreamer V3 实现了"一套超参数跨越大量任务"的通用性，是交互型范式目前最成熟的代表。
- **DayDreamer**（Wu et al., 2022）：将 Dreamer 移植到真实机器人（A1 四足机器人、XArm 机械臂、UR5 工业臂），在真实物理世界实现 online learning。机器人一边在真实环境中采集数据，一边在 replay buffer 里更新 world model，再在 world model 的想象空间里训练策略——真实交互数据极其稀缺，但想象训练可以无限进行。

DayDreamer 的思路：用少量真实摔倒数据建立世界模型，然后让机器人在想象里无限练习——不需要每次探索都付出真实物理代价。

**数学形式**：模型学习的是带动作条件的转移分布

$$p(s_{t+1} \mid s_t, a_t)$$

其中 $s_t$ 是从 $o_t$ 编码出的潜在状态。

**适用场景**：
- 机器人控制（在真实世界或仿真器中）
- 游戏智能体（Atari、DMControl等）
- 任何需要智能体主动决策的任务

**局限性——现实差距（reality gap）**：虽然相比 model-free 方法已大幅降低与环境的交互量，但在真实机器人上，数据采集仍然昂贵且危险。更深层的挑战在于：接触动力学、摩擦系数、软体形变这些在仿真器里很难完美复现。世界模型从仿真迁移到真机时，模型的误差会在规划步骤中被放大，导致策略在真实环境中失效。

### 范式三：反事实型（Counterfactual）

**核心思想**：学习"如果智能体执行了动作X，会发生什么"——即显式或隐式地建模反事实推理。模型不必生成逼真的像素观测，只需在抽象的价值或奖励层面做出正确预测。

**代表系统**：
- **MuZero**（Schrittwieser et al., 2020）：不预测像素，只学习一个隐式模型使 MCTS 搜索能找到最优策略。在围棋、国际象棋、Atari 上达到超人水平。

> **📖 MCTS（Monte Carlo Tree Search，蒙特卡洛树搜索）**：一种通过模拟"想象对弈"来评估动作优劣的规划算法。从当前状态出发，反复进行四个步骤：①**选择**：沿树向下，选择 UCB 分数最高的节点；②**扩展**：在叶节点尝试一个新动作，扩展一个新节点；③**模拟/评估**：用神经网络估计新节点的价值（MuZero 直接用 value head，不再做随机 rollout）；④**反向传播**：将价值估计沿路径向上更新。重复数百次后，访问次数最多的动作就是"经过充分搜索认为最优的动作"。
- **TD-MPC**（Hansen et al., 2022）：用潜在一致性损失训练隐式动力学，结合 MPC 规划和时序差分学习。

国际象棋大师不会在脑中"渲染"棋盘的每个像素——他在内心推演的是"如果我走马，对方可能应以炮，然后……"他的世界模型是高度抽象的战术图谱，不是棋盘照片。这正是反事实型范式的核心：用抽象表示做决策，而不是重建视觉细节。

**MuZero 的三个预测头**：MuZero 的隐式世界模型并不重建像素，而是维护三个预测头：

| 预测头 | 预测目标 | 作用 |
|--------|----------|------|
| reward head | 即时奖励 $r_t$ | 评估当前步的好坏 |
| value head | 未来累计价值 $V(s_t)$ | 引导 MCTS 搜索方向 |
| policy prior | 动作概率分布 $\pi(a \mid s_t)$ | 减少 MCTS 需要搜索的分支数 |

只要这三个预测头准确，潜在状态 $s_t$ 长什么样并不重要。这给了世界模型一个重要启示：**对 agent 来说，"真实还原世界"不一定是最优目标。** 一个为决策服务的世界模型，只需要编码对决策有用的信息。

**TD-MPC 的核心洞察**：不必重建像素，只预测"对规划有用的量"。潜在一致性损失（latent consistency loss）让编码器和动力学函数保持自洽——编码器输出的 $z_{t+1}$ 应该和动力学模型从 $z_t, a_t$ 预测的 $\hat{z}_{t+1}$ 一致。这个设计使模型可以在没有解码器的情况下端到端训练。

**数学形式**：模型学习的不是生成性的 $p(o_{t+1})$，而是价值函数与奖励的预测

$$\hat{r}_t = r(s_t, a_t), \quad \hat{V}_t = V(s_t)$$

使得基于模型的规划能够最大化长期回报。

**适用场景**：
- 复杂策略游戏与长程规划
- 对样本效率要求极高的场景
- 不需要像素级重建、只关心决策质量的任务

**局限性**：可解释性弱，难以直接可视化模型"想象"的未来；隐式模型的泛化边界不透明，在训练分布之外的行为难以预测。

### 进阶：第四种范式——World Action Model（WAM）

> 这是 2025—2026 年最前沿的方向，了解即可，不影响后续项目学习。

前三种范式分别回答了"看视频能学到什么""交互学到什么""反事实推理学到什么"。但有一个更激进的问题：**视频和动作，是不是同一个物理过程的两个侧面？**

World Action Model（WAM）试图同时预测未来世界状态和未来动作，将世界模型与行为策略统一到同一个框架下。

```
World Model:  observation + action  → future observation/state
VLA:          observation + language → action
WAM:          observation + language → future observation + action
```

**WAM 的核心意义**：视频提供了 dense physical supervision——每一帧都隐含着物体的运动、力的作用、动作的后果。WAM 利用这些监督信号，让 policy 学习物理运动和动作后果，而不只是做 action regression（从状态直接映射到动作的监督学习）。

**代表作**：
- **DreamZero**（2025）：在想象空间中联合预测世界状态和策略动作，实现零样本迁移。
- **Motus**（2025）：通过视频预测作为 dense supervision 训练机器人策略，显著提升数据效率。
- **DreamDojo**（2026）：将 WAM 框架扩展到多智能体和多具身（embodiment）场景。

WAM 代表了世界模型研究的一个根本性转向：从"为了理解世界而建模"，转向"为了行动而理解世界"。

### 三范式对比一览

| 维度 | 观察型 | 交互型 | 反事实型 |
|------|--------|--------|----------|
| 数据来源 | 离线视频 | 在线交互 | 在线交互 |
| 学习对象 | 视觉动态 | 动作-状态转移 | 价值/奖励预测 |
| 是否生成像素 | 是 | 通常是 | 否 |
| 典型代表 | Sora, Genie | Dreamer, DayDreamer | MuZero, TD-MPC |
| 适用场景 | 生成、增强 | 控制、机器人 | 策略、规划 |
| 数据规模 | 互联网级 | 有限交互 | 有限交互 |
| counterfactual | 不支持 | 支持（显式） | 支持（隐式） |

## Part B：规划机制（再问"怎么做"）

有了世界模型，智能体如何用它来选择动作？这一部分介绍三种规划机制，从最直觉的随机搜索，到 Dreamer 的想象训练，再到 TD-MPC 的混合方案。

---

### 机制一：CEM射击法 MPC（Cross-Entropy Method MPC）

> **📖 MPC（Model Predictive Control，模型预测控制）**：一种规划框架，核心思想是"**滚动时域优化**"——在每个时间步，用模型向前预测未来 $H$ 步，选出最优的动作序列，只执行第一个动作，然后在下一步重新规划。这样的好处是：即使模型不完美，频繁重规划可以及时纠正误差，不会让错误无限累积。

> **📖 累计奖励与折扣因子 γ**：智能体的目标是最大化**折扣累计奖励** $R = \sum_{h=0}^{H-1} \gamma^h \cdot r_{t+h}$，其中 $\gamma \in [0, 1)$ 是折扣因子（通常取 0.99）。$\gamma < 1$ 使近期奖励比远期奖励更重要——直觉上，"现在得到 1 分"比"10 步后得到 1 分"更有价值，因为远期预测更不确定。$\gamma$ 越接近 1，越"有远见"；$\gamma = 0$ 只关注当前奖励（极度短视）。

**一句话描述**：随机采样一批动作序列，在模型中"想象"执行，选出预期回报最高的序列，只执行第一步，然后重复。

**算法步骤（伪代码）**：

```
CEM-MPC 规划循环（每步执行一次）

输入：当前状态 s_t，世界模型 f，奖励模型 r，规划步数 H，精化轮数 K

1. 初始化动作分布：μ ← 0, σ ← 1  （均值为0，标准差为1的高斯）

2. FOR k = 1 to K（精化轮）：
   a. 从 N(μ, σ²) 采样 N 条动作序列：{a^(i)_{t:t+H}}，i=1..N
   b. FOR 每条序列 i：
        展开想象轨迹：s^(i)_{t+1} = f(s_t, a^(i)_t), ..., s^(i)_{t+H}
        计算累计奖励：R^(i) = Σ_{h=0}^{H-1} γ^h · r(s^(i)_{t+h}, a^(i)_{t+h})
   c. 选出 Top-K 条序列（按 R^(i) 降序）
   d. 用 Top-K 序列重新拟合：μ ← mean(Top-K), σ ← std(Top-K)

3. 执行 μ 的第一个动作：a_t ← μ[0]

4. 观测真实环境，得到 s_{t+1}，滑动窗口重复规划
```

**为什么要多轮精化？**

第一轮采样覆盖范围广但精度低，找到"高回报区域大概在哪里"；后续几轮用精英序列重新拟合分布，让采样范围逐渐向高回报区域收缩。迭代轮数越多，找到的动作序列越接近局部最优——代价是每步决策的计算时间线性增长。

**关键超参数**：

| 超参数 | 含义 | 取值影响 |
|--------|------|----------|
| $H$（规划步数）| 向前预测多少步 | 越大越能预见远期后果，但累积误差越高 |
| $N$（候选序列数）| 每轮采样多少条动作序列 | 越多越接近全局最优，但计算越慢 |
| Top-K 比例（精英比例）| 保留多少比例的优秀序列 | 比例太高收敛慢，太低容易陷入局部最优 |
| $K$（精化轮数）| 迭代精化多少轮 | 与 $N$ 共同决定每步决策的计算开销 |

**局限性——高维动作空间的瓶颈**：在高维连续动作空间（例如机械臂同时控制7个关节），随机搜索的效率极低。假设每个关节各有10个候选值，7个关节的联合搜索空间就有 $10^7 = 1000$ 万种组合。这是 TD-MPC 要解决的核心问题：用 Q 函数引导搜索，而不是盲目采样。

**优点**：简单、无梯度、易于实现，对世界模型的可微分性没有要求。  
**缺点**：在高维动作空间中计算开销大；短视（规划步数受限于模型误差累积）。

### 机制二：潜在空间中的 Actor-Critic（Dreamer 的做法）

> **📖 Actor-Critic 架构**：强化学习中的一种经典框架，由两个网络组成：
> - **Actor（演员）**：即策略网络 $\pi_\theta(a|s)$，输入当前状态，输出应执行的动作（或动作的概率分布）。它负责"决策"。
> - **Critic（评论家）**：即价值网络 $V_\phi(s)$，输入状态，输出该状态的预期累计奖励（即"这个状态有多好"）。它负责"评估"。
>
> 两者相互配合：Actor 采取动作，Critic 评估这些动作带来的价值，并给 Actor 提供训练信号（"这个动作比平均水平好多少"）。相比纯策略梯度方法，Critic 提供的基准（baseline）大幅降低了梯度估计的方差，使训练更稳定。

Dreamer 的核心洞察：与其在真实环境里收集大量数据训练策略，不如在**世界模型的想象轨迹**中训练——速度快、无风险、可微分。

**Dreamer 规划循环示意图**：

```
真实环境
    │
    │ (少量真实交互)
    ▼
┌─────────────────────────────────────────────────────┐
│                   世界模型（RSSM）                    │
│                                                     │
│  o_t ──[编码器]──► z_t ──[RSSM]──► z_{t+1} ──► ... │
│                         ▲                           │
│                         │ 动作 a_t                  │
│                    [Actor π]                        │
│                         ▲                           │
│                    最大化 V(z_t)                     │
│                    [Critic V]                       │
│                         ▲                           │
│              用想象轨迹的TD目标训练                    │
└─────────────────────────────────────────────────────┘
    │
    │ (Actor 输出动作作用于真实环境)
    ▼
真实环境（下一步）
```

**训练流程**：
1. **想象展开**：从当前潜在状态 $z_t$ 出发，用 Actor 采样动作，RSSM 滚动预测 $H$ 步，得到想象轨迹 $\{z_t, z_{t+1}, \ldots, z_{t+H}\}$。
2. **Critic 估值**：对每个想象状态计算 $V(z_h)$，用 $\lambda$-return 构造训练目标。
3. **Actor 优化**：Actor 通过反向传播（跨越整条想象轨迹）最大化 Critic 预测的累计价值。
4. **世界模型更新**：用真实环境数据（重建损失 + KL）更新 RSSM 和编码器。

**λ-return 的直觉**：纯蒙特卡洛需要等 episode 结束才能得到真实回报，方差高；纯 TD 只看一步，偏差大。λ-return 是两者的插值——用"前 $k$ 步真实奖励 + 第 $k+1$ 步 Critic 估值"构造 $k$ 步回报，再对所有 $k$ 加权平均。$\lambda \to 1$ 信任真实 rollout，$\lambda \to 0$ 信任 Critic。

**为什么可微分很重要**：Actor 的梯度直接从 RSSM 的可微分动力学中流过，就像在"数字世界"里做反向传播。这比蒙特卡洛采样估计策略梯度精确得多——想象一下，用有限差分估计一个复杂函数的梯度，和直接用自动微分求导，两者的精度和效率天壤之别。

**模型漏洞（Model Exploitation）问题**：

这是所有 model-based RL 面临的经典挑战。policy 可能找到模型里高奖励但真实世界不成立的动作——例如用某种高频抖动动作在世界模型里获得高分，但这个动作在真机上只会损坏电机。世界模型的误差给了策略可乘之机：只要模型有漏洞，策略就会"钻漏洞"而不是"学真本事"。

Dreamer 系列的解决思路是定期用真实环境数据更新世界模型，同时限制想象展开的步数（减少误差积累），但这个问题并没有被根本解决。

---

### 机制三：TD-MPC——两者的桥梁

TD-MPC（Temporal Difference Model Predictive Control）不选边站——它同时拥有 MPC 的前瞻规划能力和 Actor-Critic 的时序差分学习效率。

**核心设计**：

| 组件 | 作用 |
|------|------|
| 潜在一致性损失 | 训练隐式动力学模型：$\hat{z}_{t+1} = f(z_t, a_t)$ 应与编码器输出的 $\text{sg}(z_{t+1})$ 一致 |
| 时序差分目标 | 用 Bellman 方程更新 Q 函数：$Q(z_t, a_t) = r_t + \gamma \cdot Q(z_{t+1}, \pi(z_{t+1}))$ |
| CEM 规划 | 在每步决策时，用 CEM 在潜在空间中搜索最优动作序列 |

**stop-gradient 的作用**：注意一致性损失中的 `sg(z_{t+1})`——`sg` 表示 stop-gradient（停止梯度）。这个细节至关重要：如果编码器的两端（预测端和目标端）都可以被梯度更新，模型可能学到一个"恒等函数"——把所有状态映射到同一个点，使得一致性损失为零，但毫无意义。stop-gradient 固定住目标端，防止这种模式坍塌（mode collapse）。

> **💡 直觉**：这和 BYOL（Bootstrap Your Own Latent）的设计思路如出一辙。自监督学习中，防止表征坍塌是一个反复出现的工程难题，stop-gradient 是最简洁的解决方案之一。

**TD 学习 vs 纯展开的区别**：纯展开的 MPC 需要把想象轨迹一直滚到任务结束（或足够远的未来）才能得到有意义的回报估计，这要求世界模型在长序列上保持精度——而模型误差会随步数指数增长。

> **📖 Bellman 方程**：贝尔曼方程是价值函数的递归定义：$Q(s_t, a_t) = r_t + \gamma \cdot \max_{a'} Q(s_{t+1}, a')$。它说的是"在状态 $s_t$ 执行动作 $a_t$ 的价值"等于"即时奖励 $r_t$"加上"折扣后的下一步最优价值"。这个递推关系把无限步的累积奖励问题，转化为只看"一步奖励 + 下一步 Q 值"的形式，大幅降低了计算难度。
>
> **📖 自举（bootstrapping）**：在强化学习中，"bootstrapping"指用模型自身的估计（如 $Q(s_{t+1}, a')$）来作为训练目标——即"用自己预测自己"。TD 学习正是通过 Bellman 方程做 bootstrapping：$Q(s_t, a_t)$ 的目标值用 $Q(s_{t+1}, \cdot)$ 来计算，而不等到轨迹结束才得到真实的总回报。这使学习可以在每步都进行，无需等到 episode 结束，但也引入了偏差（因为目标本身不准确）。

TD 学习用 Bellman 方程，以"当前奖励 + 下一步 Q 值估计"代替完整展开，将有效规划深度从"模型精确步数"缩短到"1步 + Q函数的自举"。

**与 DreamerV3 的对比**：

| 维度 | DreamerV3 | TD-MPC2 |
|------|-----------|---------|
| 设计哲学 | 一套超参数跨越大量任务 | 跨任务、跨 embodiment 的可扩展性 |
| 世界模型形式 | 显式生成（重建像素/观测） | 隐式（只保证价值预测准确） |
| 规划方式 | 潜在空间 Actor-Critic | CEM + TD |
| 适用任务范围 | 视觉复杂任务，需要丰富观测 | 状态观测任务，高效连续控制 |
| 可解释性 | 可以可视化重建 | 潜在空间无直接语义 |

### 三种规划机制对比

| 维度 | CEM-MPC | Dreamer Actor-Critic | TD-MPC |
|------|---------|---------------------|--------|
| 规划方式 | 随机搜索 | 策略梯度（可微） | 随机搜索 + TD |
| 是否需要像素重建 | 否 | 是 | 否 |
| 长程规划能力 | 受 $H$ 限制 | 靠 Critic bootstrap | TD + MPC 结合 |
| 计算成本 | 高（大 $N$） | 中（想象展开） | 低-中 |
| 高维动作空间 | 效率低 | 梯度直接优化 | Q函数引导搜索 |
| 模型漏洞风险 | 中（短视） | 高（policy可钻漏洞） | 中（TD抑制累积误差） |
| 典型场景 | 简单连续控制 | 视觉复杂任务 | 高效连续控制 |

## 本讲小结

- **三范式**决定了世界模型"用什么数据、学什么知识"：观察型学视觉规律但无法控制，交互型学动作因果但数据昂贵，反事实型学价值推理但可解释性弱。
- **WAM（第四范式）**代表最新趋势：将世界预测与动作生成统一，视频作为 dense physical supervision 训练具身策略。
- **三规划机制**决定了世界模型"如何被用于决策"：CEM 最直白但在高维空间效率低，Actor-Critic 最优雅但有模型漏洞风险，TD-MPC 最务实地兼顾了两者。
- Dreamer = 交互型范式 + 潜在 Actor-Critic 规划，是本系列课程的核心参考系统。
- TD-MPC = 反事实型范式 + CEM + TD，将在 P04 中亲手实现并与 Dreamer 对比。
